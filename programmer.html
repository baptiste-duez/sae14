Niveau 1 — S’intégrer dans un service informatique

Ce niveau regroupe les compétences essentielles pour comprendre un environnement informatique, manipuler des outils de développement, collaborer efficacement et participer à la création d’applications.

AC13.01 — Utiliser un système informatique et ses outils

⚡ Ce que j’ai fait

Prise en main des environnements Linux, gestion des répertoires en ligne de commande et configuration d'éditeurs de code (VS Code).

🛠️ Comment je l'ai fait ?

Manipulations pratiques en cours de base des systèmes d'exploitation (R1.08) et applications durant la SAE1.01.

🎯 Pourquoi je l'ai fait ?

Afin de posséder un espace de travail structuré, maîtriser le terminal d'exécution et respecter les protocoles de sécurité informatique de base.

⚠️ Mes difficultés

La mémorisation de la syntaxe des commandes d'administration système et la gestion des arborescences de droits de fichiers.

💡 Ce que j'ai appris

Naviguer rapidement au clavier, automatiser les déplacements de fichiers et appliquer les règles essentielles d'hygiène informatique.

🔄 Amélioration

Gagner en rapidité sur l'édition de fichiers de configuration textuels directement depuis un terminal distant sans interface graphique (via Nano ou Vim).

📁 Preuves

Guide des commandes shell fondamentales.

AC13.02 — Lire, exécuter, corriger et modifier un programme

⚡ Ce que j’ai fait

Analyse de scripts Python existants, traçage des variables étape par étape, correction de bugs de syntaxe et logique d'exécution.

🛠️ Comment je l'ai fait ?

Exercices de débogage et de réécriture de code lors des séances de TP en R1.07 (Fondamentaux de la programmation).

🎯 Pourquoi je l'ai fait ?

Afin de développer une capacité de lecture d'un code externe et savoir corriger efficacement des scripts d'automatisation réseau défectueux.

⚠️ Mes difficultés

Comprendre l'origine des erreurs de types cachées (erreurs logiques entre types chaînes de caractères et entiers) lors des saisies utilisateurs.

💡 Ce que j'ai appris

Analyser l'historique d'erreurs (traceback) renvoyé par l'interpréteur Python pour cibler instantanément la ligne problématique.

🔄 Amélioration

Prendre le réflexe d'intégrer des commentaires détaillés lors de chaque modification pour faciliter la relecture par d'autres collaborateurs.

📁 Preuves

Scripts Python corrigés et optimisés avec rapports de tests d'exécution.

AC13.03 — Traduire un algorithme dans un langage donné

⚡ Ce que j’ai fait

Conception de structures algorithmiques (boucles, conditions, fonctions) et traduction en instructions de code Python structurées.

🛠️ Comment je l'ai fait ?

Création d'algorithmes théoriques sur papier puis développement de scripts d'applications de calculs en R1.07.

🎯 Pourquoi je l'ai fait ?

Pour acquérir une logique de développement universelle permettant de découper un problème complexe en une suite d'instructions simples pour la machine.

⚠️ Mes difficultés

Gérer de manière optimisée l'imbrication de boucles complexes (conditions `if/while`) sans compromettre la lisibilité globale du script.

💡 Ce que j'ai appris

Utiliser les listes, dictionnaires et fonctions pour modulariser un programme et rendre le traitement des données dynamique.

🔄 Amélioration

Savoir concevoir des organigrammes clairs avant de commencer la rédaction du code pour éviter les erreurs de logique structurelle.

📁 Preuves

Algorithmes sources et codes sources des programmes applicatifs développés en Python.

AC13.04 — Connaître l’architecture et les technologies d’un site Web

⚡ Ce que j’ai fait

Création et mise en page d'interfaces web statiques, structuration du squelette sémantique et application de styles graphiques responsives.

🛠️ Comment je l'ai fait ?

Écriture manuelle de code en HTML5 et CSS3 dans le cadre du module Introduction aux technologies Web (R1.09).

🎯 Pourquoi je l'ai fait ?

Pour comprendre les mécanismes d'échanges de la couche application (requêtes HTTP/HTTPS entre navigateur client et serveurs web Apache/Nginx).

⚠️ Mes difficultés

Assurer le positionnement d'affichage multi-écrans (adaptation mobile/tablette) via les propriétés Flexbox ou CSS Grid.

💡 Ce que j'ai appris

Structurer une page de manière propre, lier des feuilles de style externes et analyser le code source via l'outil d'inspection du navigateur.

🔄 Amélioration

Approfondir la programmation JavaScript côté client afin de rendre les composants de mes pages web pleinement interactifs.

📁 Preuves

Ce site en regardant le code source on voit qu'il est bien structurer.

AC13.05 — Choisir les mechanisms de gestion de données adaptés

⚡ Ce que j’ai fait

Extraction, tri et traitement de flux de données brutes stockés sous forme de fichiers structurés (formats CSV ou JSON).

🛠️ Comment je l'ai fait ?

Développement de scripts d'automatisation de lecture et d'analyse statistique dans le cadre de la SAE1.05 (Traiter des données).

🎯 Pourquoi je l'ai fait ?

Pour être en mesure d'analyser des données systèmes massives (fichiers de logs de serveurs) afin d'en extraire des indicateurs exploitables.

⚠️ Mes difficultés

Le nettoyage des lignes de données corrompues ou incomplètes qui provoquaient des erreurs fatales lors de l'exécution des boucles de tri.

💡 Ce que j'ai appris

Ouvrir et fermer dynamiquement des flux de fichiers de données externes en code et automatiser des calculs statistiques (moyennes, maximums).

🔄 Amélioration

Évoluer vers l'utilisation de bases de données relationnelles (langage SQL) pour gérer le stockage de manière plus performante et sécurisée.

📁 Preuves

Rapport d'analyse de données et scripts de traitement automatique développés pour la SAE1.05.
https://gitlab.univ-artois.fr/baptiste_duez/sae15

AC13.06 — S’intégrer dans un environnement propice au développement

🔄 Amélioration

Intégrer les systèmes de contrôle de version distribués (comme Git et GitHub) dans mes futurs projets pour fluidifier la fusion des codes d'équipe.

📁 Preuves

Dossier d'organisation de projet et livrables de suivi d'équipe réalisés en SAE1.04.
